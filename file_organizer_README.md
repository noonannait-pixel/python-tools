# 🔮 File Organizer — 批量文件整理/重命名工具

## 📌 工具名称
**File Organizer（批量文件整理助手）**

## 🎯 解决什么问题
日常工作和学习中，我们经常遇到桌面或文件夹里文件混乱的情况：

- 一堆 `.docx`、`.xlsx`、`.jpg` 混在一起，找文件像大海捞针
- 需要批量给文件加前缀/后缀（比如统一加「项目_」）
- 希望按修改时间给文件排序重命名
- 手动一个一个改，费时又容易出错

这个工具就是解决这些痛点的——**一键批量整理，安全可靠，操作前可预览，执行后有日志**。

---

## ✅ 已实现功能

| 功能 | 说明 | 状态 |
|------|------|:----:|
| 📂 按类型分类 | 按文件后缀自动归类到子文件夹（图片/文档/表格/演示等） | ✅ |
| ➕ 添加前缀 | 批量给所有文件名添加自定义前缀 | ✅ |
| ➕ 添加后缀 | 批量给所有文件名添加自定义后缀（扩展名前） | ✅ |
| 🕒 按修改时间重命名 | 按文件修改时间排序，统一重命名（格式：YYYYMMDD_HHMMSS） | ✅ |
| 👁️ 预览模式 | 所有操作先显示「原名 → 新名」对比，确认后才执行 | ✅ |
| 🛡️ 安全过滤 | 自动跳过 log.txt、脚本自身、临时文件等 | ✅ |
| 📝 日志记录 | 每次执行自动生成 log.txt，记录原路径、新路径、操作时间、操作类型 | ✅ |
| 🔄 防冲突 | 新文件名已存在时自动加 _1、_2 后缀，不覆盖原文件 | ✅ |
| 🔒 目录保护 | 禁止操作桌面、下载、系统等关键目录 | ✅ |
| 🧪 测试模式 | 首次运行自动创建 test_files 文件夹并生成15个测试文件 | ✅ |

## 📖 使用方法

### 运行方式
```bash
# 双击运行，或在命令行执行：
python file_organizer.py
```

### 操作步骤
1. **启动程序** — 自动打开 test_files 测试文件夹
2. **浏览文件** — 文件列表显示所有文件名、大小、类型、修改时间
3. **选择操作** — 点击「按类型分类」「添加前缀」「添加后缀」或「按修改时间重命名」
4. **预览结果** — 下方预览区显示「原名 → 新名」对比
5. **确认执行** — 点击「✅ 确认执行」，真正修改文件
6. **查看日志** — 自动生成 log.txt，记录完整操作详情

### 测试流程
```
首次运行 → 自动创建 test_files → 有15个测试文件
→ 点「添加后缀」输入 "_v2" → 预览 → 确认执行
→ 文件全部改为 xxx_v2.xxx → log.txt 记录操作
→ 验证 log.txt 未被修改 → ✅ 安全
```

## 🔒 安全设计

```
┌─────────────────────────────────────────┐
│           五重安全保障                   │
├─────────────────────────────────────────┤
│ 1. 禁止操作系统目录                      │
│    - Desktop / Downloads / Documents    │
│    - C:\Windows / Program Files        │
│                                          │
│ 2. 先预览后执行                          │
│    - 不点「确认执行」绝不碰文件           │
│                                          │
│ 3. 跳过保护文件                          │
│    - log.txt / .py / .tmp / 隐藏文件    │
│                                          │
│ 4. 文件名冲突自动加 _1 _2                │
│    - 绝不覆盖任何已有文件                 │
│                                          │
│ 5. 完整操作日志                          │
│    - log.txt 可追溯每一次修改            │
│    - 原路径 → 新路径 → 操作时间           │
└─────────────────────────────────────────┘
```
